Technical Specifications

Aspect WSL Setup MCP Setup
Protocol Direct system calls JSON-RPC 2.0 over stdio
Communication Native process execution Structured message passing
File Access Full filesystem (Linux + Windows mounts) Depends on server implementation
Performance Native speed, minimal overhead JSON serialization overhead
Security Model WSL user permissions Server-defined permissions
Configuration None required ~/.config/claude/mcp_settings.json
Extensibility Limited to built-in tools Unlimited via custom servers
Resource Access Local system only Any resource the server can reach

Available Tools Comparison

WSL Built-in Tools

  • Bash - Execute shell commands
  • Read - Read file contents
  • Write - Create/overwrite files
  • Edit - Modify file contents
  • MultiEdit - Batch file edits
  • Glob - Find files by pattern
  • Grep - Search file contents
  • LS - List directory contents
  • WebFetch - Fetch web content
  • WebSearch - Search the web

MCP Server Examples

  • postgres - PostgreSQL queries
  • github - GitHub API access
  • slack - Slack integration
  • puppeteer - Browser automation
  • filesystem - Enhanced file ops
  • sqlite - SQLite database
  • git - Advanced git operations
  • docker - Container management
  • aws - AWS service access
  • Custom servers for any API/service

Use Case Scenarios

When to Use WSL (Your Current Setup)

  • Local development projects
  • File-intensive operations
  • Quick prototyping
  • Standard web development
  • Git operations
  • Running build tools
  • Direct system access needed
  • Maximum performance required

When to Add MCP

  • Database integration needed
  • External API access
  • Cloud service integration
  • Custom tool requirements
  • Team-shared configurations
  • Sandboxed access preferred
  • Cross-platform consistency
  • Specialized workflows

Configuration Examples

Current WSL Setup

# No configuration needed!
# Just run:
claude

# Access your project:
cd /mnt/c/projects/road_mapper
npm install
npm run dev

MCP Configuration

# ~/.config/claude/mcp_settings.json
{
  "servers": {
    "postgres": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-postgres"],
      "env": {
        "POSTGRES_URL": "postgresql://..."
      }
    },
    "github": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "ghp_..."
      }
    }
  }
}

Summary

Your current WSL setup is ideal for the Road Mapper project because:

  • You have direct file system access
  • Native performance for development tasks
  • All necessary tools are built-in
  • No additional configuration needed

Consider adding MCP when you need:

  • Database integration for storing road maps
  • Cloud deployment automation
  • External API integrations
  • Team collaboration features
